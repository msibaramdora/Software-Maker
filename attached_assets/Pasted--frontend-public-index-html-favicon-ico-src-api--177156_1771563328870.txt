## **Routes Structure**

```javascript
// Public Routes
/                                    → HomePage
/login                               → LoginPage (Employee)
/visitor/register                    → VisitorRegistration (QR scan at gate)
/visitor/invitation/:token           → InvitationAccept (Email link)

// Employee Routes (Protected)
/employee/dashboard                  → EmployeeDashboard
/employee/send-invitation            → SendInvitation
/employee/requests                   → ApprovalPage (Approve/Reject)

// Watchman Routes (Protected)
/watchman/dashboard                  → WatchmanDashboard (Stats)
/watchman/scan                       → ScanQRPage
/watchman/visitor/:id                → VisitorDetailsPage (After scan)
```

## **User Flows**

### **Flow 1: Invitation-based Registration**

1. Employee logs in → `/login`
2. Employee sends invitation → `/employee/send-invitation`
3. Visitor receives email with link
4. Visitor clicks link → `/visitor/invitation/:token`
5. Visitor fills form with photo → Submit
6. Email sent to employee & visitor (pending)
7. Employee approves → `/employee/requests`
8. Visitor receives approval email with gate pass

### **Flow 2: Direct Registration at Gate**

1. Visitor scans QR at gate → `/visitor/register`
2. Visitor fills form with photo → Submit
3. Email sent to employee & visitor (pending)
4. Employee approves → `/employee/requests`
5. Visitor receives approval email with gate pass

### **Flow 3: Watchman Check-in/out**

1. Watchman scans visitor's QR → `/watchman/scan`
2. Shows visitor details & validity
3. If valid → Check-in button (saves time)
4. Visitor leaves → Scan again → Check-out button
5. Dashboard shows: today visits, in office, left
